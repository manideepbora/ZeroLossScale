The 7-Step Protocol

Walkthrough: Scale 2 → 3 Partitions

Before (steady state)

KV: count=2, mode=direct order-A → hash%2=0 → Consumer-0 order-B → hash%2=1 → Consumer-1

Step 1-3: Buffer & Drain

KV: count=2, mode=buffer order-C → Buffer Stream order-D → Buffer Stream Consumer-0 draining... Consumer-1 draining...

Step 4-5: Update & Scale

KV: count=3, mode=buffer Consumer-0 ready Consumer-1 ready Consumer-2 NEW

Step 6-7: Restore & Replay

KV: count=3, mode=direct order-E → hash%3=0 → Consumer-0 Replay: C→P.1 D→P.2 Zero lost. Zero dupes.

Technical Deep Dive

Producer Protocol (Any Language)

// Startup local_mode = kv.get("mode") // "direct" or "buffer" local_count = kv.get("partition_count") // Async watch — react to changes kv.watch("mode", v => local_mode = v) kv.watch("partition_count", v => local_count = int(v)) // Publish function publish(msg): if local_mode == "buffer": js.publish(buffer_subject, msg) else: partition = hash(msg.key) % local_count js.publish(prefix + "." + partition, msg)

~15 lines. Go, Python, Java, Rust, Node — any language with a KV client.

KV Signaling vs Traditional RPC

RPC Coordination

Coordinator → RPC → Producer 1 Coordinator → RPC → Producer 2 Coordinator → RPC → Producer 3 (down!) BLOCKED — custom SDK required

KV Watch

Coordinator → KV: mode=buffer ↓ (watch) Producer 1 reacts Producer 2 reacts Producer 3 (down — no problem)
No coupling. No custom SDK. Producers join/leave freely. Down producer doesn't block repartition.

Crash Recovery: Leader Failover

Replica A (leader) Replica B (standby) | | Step 1: Buffer ✓ | Step 2: Wait ✓ | Step 3: Drain ✓ | ✗ CRASH | | elected leader | reads KV: step=3 | skips 1-3, resumes: | Step 4 ✓ | Step 5 ✓ | Step 6 ✓ | Step 7 ✓ DONE
Each step checkpointed to KV. Any replica resumes from exact failure point. Every step is idempotent — safe to re-execute.

Auto-Scaler: Lag-Based Decisions

Loop (leader only, every 10s): total_lag = sum(num_pending + num_ack_pending) across all consumers if total_lag > high_watermark AND current < max: KV: desired_partitions = current + 1 → Reconciler triggers scale-up if total_lag < low_watermark AND current > min: KV: desired_partitions = current - 1 → Reconciler triggers scale-down cooldown: 30s between decisions → prevents oscillation
Auto-scaler writes desired_partitions to KV. Reconciler watches and triggers the Scaler. Separation = stateless, restartable.

DLQ + Retries

  • Failed messages → DLQ stream with metadata
  • KV writes: 3 retries, 500ms backoff
  • Consumer scaling: 3 retries, 1s backoff
  • Replay publish: idempotent msg IDs
  • All retries exhausted → revert to direct mode
